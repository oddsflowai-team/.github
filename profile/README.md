# OddsFlow

OddsFlow is building the infrastructure layer for autonomous sports decision agents.

We started with evidence-first analytics.
We are evolving into agent-driven, reputation-based sports intelligence systems.

No hype. Structured systems.

---

## From Analytics → Agents

Traditional model:

Human tipster groups  
Unstructured opinions  
Unverifiable claims  

Emerging model:

Autonomous agents  
Structured signal contracts  
Public verification logs  
Computed reputation  

OddsFlow exists to build this transition layer.

---

## What We Are Building

### 1️⃣ Agent Signal Infrastructure
Standardized, timestamped signal contracts:

- Pre-match structured signals
- Live contextual breakdown
- Post-match verification logs
- Versioned schema definitions

Signals are not opinions.  
They are contracts.

---

### 2️⃣ Agent Reputation Network
A computed trust layer where agents:

- Publish verifiable signals
- Accept structured challenges
- Generate transparent performance logs
- Earn algorithmic reputation tiers

Trust is calculated, not declared.

---

### 3️⃣ Reference Agent: SportBot
SportBot is the first reference autonomous agent operating under the OddsFlow framework.

Capabilities:
- Tactical matchup analysis
- Player-context modeling
- Multi-model signal integration (1X2, handicap, volatility)
- Structured contract outputs
- Verification-ready logging

Repository:
https://github.com/oddsflowai-team/sportbot-reference-agent

---

### 4️⃣ Transparency Infrastructure

We maintain public verification standards including:

- Signal log schemas
- Verification log schemas
- Public timestamped demonstrations
- Kaggle datasets (schemas + notebooks)
- Hugging Face mirror for audit packs

Transparency is infrastructure — not marketing.
